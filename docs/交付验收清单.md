# 安卓无中心对讲交付验收清单（RF mesh）

> 现场执行细则见 [`现场测试方案与执行手册.md`](现场测试方案与执行手册.md)

## 1. 交付边界

- 网络边界：仅 **同一射频任务密钥组成的 RF mesh**（扁平 IP 子网内互通）；不支持跨 mesh / 跨密钥域。不依赖外部 Wi-Fi/以太网基础设施。
- 保密边界：异射频密钥设备无法进同一 mesh，应用层不可发现、不可呼叫。
- 会话边界：默认单活动会话（`maxActiveSessions=1`）；组呼可配置 `maxGroupModules`（默认 5）。
- 功能范围：单呼、组呼（module 级 RTP）、PTT 抢权（Floor Owner + CAS）、挂断、忙线拒接、自动重拨、前台服务保活。
- 发现：静态 Peer（主）+ mDNS（辅）。

## V1 单呼验收

### 2. 安全与协议检查

- [ ] 信令消息包含 `nonce` 与 `signature`。
- [ ] 收包端启用签名验签（共享密钥一致）。
- [ ] 收包端启用防重放（时间窗 + nonce 去重）。
- [ ] 启用模块白名单（如有要求）。

### 3. 功能验收

- [ ] 设备发现：同 RF mesh 内两台设备可互相发现（或静态 Peer 可呼叫）。
- [ ] 保密隔离：异射频密钥的两台设备互不可见（可选，保密任务必测）。
- [ ] 单呼建立：`CALL_INVITE -> CALL_ACCEPT` 链路可用。
- [ ] PTT流程：按下请求发言、授权后讲话、松开释放；双机同时 PTT 仅一方讲话。
- [ ] 忙线拒接：占线时新呼叫收到 `CALL_REJECT(BUSY)`。
- [ ] 挂断流程：任一方挂断后双方会话清理。
- [ ] 自动重拨：目标模块恢复上线后可自动重拨（开关开启时）。

### 4. 稳定性验收

- [ ] 连续通话 30 分钟无崩溃。
- [ ] 弱网（模拟 5% 丢包）仍可懂。
- [ ] NSD 异常时可自动重试发现。
- [ ] UDP 信令 socket 异常后可自动重建。

## V2 组呼与目录验收

- [ ] HELLO 同步：对端可见远端 endpoint 在线列表。
- [ ] 组呼：`GROUP_INVITE -> GROUP_ACCEPT`，module 级单 RTP。
- [ ] 组内 PTT 抢权：CAS 裁决，无双方同时上行。
- [ ] 静态 Peer：关闭 mDNS 后仍可发现/呼叫。
