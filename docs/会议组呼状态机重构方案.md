# 会议/组呼会话状态机重构方案

> 版本：V1.0  
> 日期：2026-06-15  
> 关联：[`现场测试方案与执行手册.md`](现场测试方案与执行手册.md)

## 1. 背景

现场测试发现 M02 在会议邀请 pending 期间仍接受 GROUP 组呼，导致信令/媒体 session 竞态；`acceptPendingConferenceInvite` 先删除 pending 再入会，失败时邀请丢失；M01 UI 将邀请名单与 HELLO 在线混为「已入会」。

## 2. 目标架构

四层状态分离 + 单 channel 模式 FSM：

| 层级 | 职责 |
|------|------|
| **模式层** `ChannelModeFsm` | 每 channel：`IDLE` / `GROUP_PTT` / `CONFERENCE` 互斥，经 `requestMode()` 切换 |
| **信令层** `InviteState` | `NONE` → `INVITING` / `RINGING` → `ACCEPTED` / `DECLINED` / `EXPIRED` |
| **媒体层** `MediaState` | `NONE` → `CONNECTING` → `CONNECTED` / `RECONNECTING` / `FAILED` |
| **UI 层** | 仅 `media == CONNECTED` 计为在会；`INVITING` / `CONNECTING` / `EXPIRED` 独立展示 |

**不变量**：`connectedRemoteCount` = ICE CONNECTED 远端数；UI 人数 = `connectedRemoteCount + 1`（含本机）。

## 4. 行为变更摘要

### 4.1 邀请原子化（P0）

- `acceptGroupInvite()` 返回 `Boolean`
- `canAcceptGroupInvite()` 只读校验，不修改 session
- `acceptPendingConferenceInvite()`：校验 → yield 本地 solo 会议 → `prepareForGroupInvite` → `acceptGroupInvite` → **成功后才** `remove` pending

### 4.2 pending 期间拒 GROUP（P0）

- `blocksGroupOnChannel()`：活跃 CONFERENCE、pending 邀请、UI/FSM 会议倾向、会议结束后 15s 冷却
- `prepareForGroupInvite` / `handleGroupJoin` 统一走 `blocksGroupOnChannel`

### 4.3 pending TTL（P0）

- 被叫侧：`conferenceInvitePendingTtlMs`（默认 30s）超时 → 发送 `CALL_REJECT EXPIRED`，清理 pending
- 主叫侧：`conferenceInviteRingTimeoutMs`（默认 20s）无应答 → `InviteState.EXPIRED`，写入 `reDialByRemoteModule`

### 4.4 UI 三态（P2）

- `TalkbackSessionSnapshot.memberViews`
- `EndpointStatus` 扩展：`INVITING` / `CONNECTING` / `EXPIRED`
- 会议进行中 `onlineCount` = `connectedRemoteCount + 1`

## 5. 配置项

在 `TalkbackCoordinatorConfig`：

| 字段 | 默认 | 说明 |
|------|------|------|
| `conferenceInvitePendingTtlMs` | 30000 | 被叫 pending 邀请超时 |
| `conferenceInviteRingTimeoutMs` | 20000 | 主叫振铃超时标记未接听 |
