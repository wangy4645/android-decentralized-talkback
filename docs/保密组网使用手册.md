# Talkback 保密组网使用手册

> 版本：V1.0  
> 适用范围：路线 A（射频任务密钥 + 应用层任务群组）已实现版本  
> 关联文档：[`系统设计-安卓无中心对讲.md`](系统设计-安卓无中心对讲.md)、[`ANT-PTT与Talkback架构对比分析.md`](ANT-PTT与Talkback架构对比分析.md)、[`现场测试方案与执行手册.md`](现场测试方案与执行手册.md)

---

## 1. 文档目的

本手册面向**现场操作员与运维人员**，说明 Talkback 在 RF mesh 保密组网场景下的完整使用流程，包括：

- 首次配置与出任务标准作业程序（SOP）
- 任务群组（Task Profile）的创建与切换
- 组呼 PTT、会议、单呼的日常操作
- 安全注意事项与常见问题排查

**前提**：设备底层为 **射频自动组网（RF mesh）**。同射频任务密钥的设备组成 mesh 后获得 IP，App 在其上运行无中心对讲。

---

## 2. 两层密钥模型

保密组网依赖**两层独立密钥**，操作顺序不可颠倒。

| 层级 | 配置方式 | 作用 | App 是否管理 |
|------|----------|------|--------------|
| **射频任务密钥** | 硬件 / 带外配置工具 | **保密边界**：决定谁能进入同一 RF mesh | 否；App 中 `RF key label` 仅作备注提醒 |
| **应用层任务密钥**（sharedSecret） | App「任务群组」档案 | 信令 HMAC 完整性、防篡改、任务内 Channel 分组 | 是 |

**出任务标准流程**：

```
1. 带外切换射频任务密钥（全队一致）
2. 设备自动进入同一 RF mesh
3. App 切换到对应「任务群组」并启动服务
4. 发现对端 → 组呼 / 单呼 / 会议
```

射频密钥的生命周期（擦除、轮换）由运维流程负责，**App 不会自动擦除射频密钥**。

## 3. 首次使用（单台设备）

### 3.1 配置设备身份

路径：**Settings → My Identity**

| 项 | 说明 | 示例 |
|----|------|------|
| moduleId | 通信模块全局唯一 ID | `M01` |
| endpointId | 本模块终端 ID | `E01` |
| Channel ID | 频道标识 | `CH-01` |
| Channel display name | 显示名称 | `Alpha Team` |

### 3.2 配置任务群组（必做）

路径：**Settings → Task Groups**

首次启动时，App 会从当前配置 **自动迁移一个默认任务群组档案**。每个档案包含：

| 字段 | 说明 |
|------|------|
| Task group name | 任务名称，如「任务 Alpha」 |
| Shared secret | **应用层任务密钥（必填，不能为空）** |
| Channel ID / display name | 本组频道信息 |
| RF key label | 射频密钥备注，如 `Key-20260610-A`（仅提示，不参与入网） |
| Manual peer override（可选） | 手动覆盖地址表，正常出任务**留空** |

**同一任务群组内所有设备**：`sharedSecret`、频道信息应保持一致。成员地址由 **自动发现** 完成，无需手填。

### 3.3 自动发现（默认，零配置）

服务启动后，App 在 RF mesh 子网上执行：

1. **子网单播探测**（`DISCOVERY_PROBE` → 统一发现端口 `51999`）
2. **HMAC 签名回应**（`DISCOVERY_ANNOUNCE`，仅同 `sharedSecret` 设备响应）
3. **Gossip 扩散**（ANNOUNCE 携带已知 peer 列表，一次接触可学到全员）
4. **HELLO** 同步 endpoint 目录（在已知地址上）

**操作员无需配置 Peer 列表**。异密钥设备验签失败，互不可见。

### 3.4 手动 Peer 覆盖（可选，排障用）

正常出任务**留空即可**。仅在自动发现失败时，用表单添加（**不要手写 JSON**）：

| 步骤 | 操作 |
|------|------|
| 1 | 在对方设备 **Settings → Service** 查看本机 IP，点 **Copy IP** |
| 2 | 本机 **Task Groups → Edit**（或 **Settings → Service**） |
| 3 | 填写 **Module ID**、**IP**、**Port** → **Add peer** |
| 4 | 保存 / 重启服务 |

示例：M01 添加 M02 → Module ID=`M02`，IP=`192.168.1.102`，Port=`50001`

手动条目优先级高于自动发现。JSON 导入仅在 **Advanced** 中供工程师批量部署，操作员无需使用。

### 3.5 启动服务

路径：**Settings → Service**

1. 确认信令端口（每台 module 不同，如 M01=50000、M02=50001）
2. 点击 **Start Service**

若 **任务密钥为空**，服务拒绝启动，提示：

> `Task shared secret is required before starting the service.`

### 3.6 安全设置（可选）

路径：**Settings → Security**

- **Shared secret**：与当前活动任务群组一致（修改会同步回档案）
- **Whitelist**：可限制仅允许指定 moduleId 通信

## 4. 出任务完整流程（推荐 SOP）

### 4.1 任务前（指挥 / 运维）

1. **确定本次任务射频密钥**（如 `TASK-ALPHA-0610`）
2. **带外下发**到所有参战通信模块（射频配置工具）
3. 在 App 中创建本次 **任务群组档案**：
   - 名称：任务 Alpha
   - Shared secret：高熵随机串（全队相同，经安全渠道下发）
   - Static peers：本次参战 module 列表
   - RF key label：填写 `TASK-ALPHA-0610`（提醒用）
4. **每台设备切换到该档案**

### 4.2 任务中（操作员）

1. 确认已带外切换射频密钥（全队同网）
2. App 切换任务群组：
   - **Settings → Task Groups → Switch**，或
   - **Talk 页 → 点击频道标题区** → 选择群组
3. 阅读确认提示后完成切换（会离开当前组网、中断通话）
4. **Settings → Service → Start Service**（若未运行）
5. 进入 **Talk** 页使用对讲

### 4.3 任务后

- 射频密钥处置按运维流程执行（App 不自动擦除）
- 可保留档案供下次复用，或删除 / 更换新密钥

## 5. 切换任务群组（多群轮流听）

### 5.1 适用场景

同一设备需要 **轮流参与不同保密任务**，类似对讲机「切信道」——**一次只听一个群**。

### 5.2 操作方式

| 方式 | 路径 |
|------|------|
| A | Settings → Task Groups → 选择档案 → **Switch** |
| B | Talk 页 → 点击频道标题区 → 选择群组 |

### 5.3 切换确认框含义

> 将离开当前群组并连接到「XXX」。进行中的通话会结束。请确保已应用匹配的射频密钥。

### 5.4 能力边界

| 能做 | 不能做 |
|------|--------|
| 保存多个档案，人为切换轮流听 | 单射频同时监听多个不同密钥的 mesh |
| 切换后自动重新发现成员 | 不切射频密钥就切 App 群组（会连不上或连错网） |

**切换 = 离开当前 RF mesh 上的应用组网**。切走后，原群组话务会 **漏听**。

### 5.5 正确切换顺序

```
1. 带外切换射频密钥 → 进入新 mesh
2. App 切换到对应任务群组
3. 若服务在运行，会自动重启并重新发现对端
```

## 6. 对讲功能使用

服务启动且发现对端后：

### 6.1 组呼 PTT（半双工，默认）

- Talk 页选择 **PTT** 模式
- **按住 PTT** 说话，松开释放话权
- 同时仅一人上行（Floor 抢权）

### 6.2 会议（全双工）

- Talk 页选择 **Meeting** 模式
- ICE 连通后默认开麦，多人可同时讲话
- **点按 PTT** = 自我静音 / 取消静音
- 规模上限：≤ 8 个 module

### 6.3 单呼

- **Contacts** 页选择对端 endpoint 发起单呼

## 7. 组网与发现说明

```
射频层（带外配置）
  同密钥 → RF mesh → 每台设备获得 IP

应用层（App）
  Gossip 子网探测（主）+ mDNS（辅）+ 手动 Peer（覆盖）→ 发现 module
  HELLO → 同步 endpoint 列表
  信令 HMAC 验签 → 建立通话
```

| 问题 | 答案 |
|------|------|
| 谁能看见谁？ | 仅 **同一射频密钥 mesh** 内的设备 |
| 异密钥设备能否互通？ | 否，应用层互不可见、不可呼叫 |
| HELLO 是否做入网门禁？ | 否，仅做端点目录同步；入网门禁在射频层 |

## 8. 版本与安全注意

1. **全员同版本升级**  
   信令已升级为 **HMAC-SHA256**，含此改动的版本与旧版 **不能互通**。

2. **禁止空密钥**  
   不得使用空 shared secret；旧默认密钥 `talkback-lan-secret` 已移除。

3. **密钥不要写入日志**  
   运维下发应使用安全渠道（二维码、NFC、加密配置包等）。

4. **切换群组会中断通话**  
   切换前请结束或接受当前通话被中断。

5. **至少保留一个任务群组**  
   最后一个档案不可删除。

## 9. 常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 无法启动服务 | 任务密钥为空 | Settings → Security 或 Task Groups 填写 shared secret |
| 看不到对端 | 射频密钥不一致 | 带外确认全队同密钥 |
| 能看到但无法通话 | 应用层密钥不一致 | 核对任务群组 shared secret |
| 切换群组后无对端 | 未切射频 / Peer 未更新 | 先切射频，再检查静态 Peer |
| 旧设备连不上新设备 | App 版本不一致 | 全部升级到含 HMAC 的版本 |
| mDNS 发现不稳定 | 多跳 mesh 组播丢包 | 依赖 Gossip 单播发现；必要时用手动 Peer 覆盖 |

## 10. 出任务前快速检查清单

- [ ] 全队射频任务密钥一致（带外确认）
- [ ] 全队 App 任务群组档案一致（名称、secret、peers）
- [ ] 每台 moduleId / endpointId / 信令端口不冲突
- [ ] shared secret 已填写且非空
- [ ] 自动发现正常（或已配置手动 Peer 覆盖）
- [ ] 服务已启动（Settings → Service 显示 Running）
- [ ] Talk 页能看到在线 module
- [ ] 所有设备为同一 App 版本
